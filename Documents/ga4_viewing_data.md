# 如何在 GA4 中查看追蹤資料

## 一、即時查看（Real-time）

### 1. 即時報表
**路徑**：「報表」→ 「即時」

**可以看到：**
- 目前正在瀏覽網站的訪客數量
- 即時頁面瀏覽
- 即時事件觸發（按鈕點擊、滾動等）
- 即時流量來源

**使用時機：**
- 測試追蹤是否正常運作
- 監控活動期間的即時流量

### 2. 即時事件
在即時報表中，點擊「事件」標籤，可以看到：
- 所有正在觸發的事件
- 事件名稱、類別、標籤
- 觸發次數

## 二、查看所有追蹤的事件

### 1. 事件總覽
**路徑**：「報表」→ 「參與度」→ 「事件」

**可以看到：**
- 所有自訂事件的列表
- 每個事件的觸發次數
- 事件參數（如標籤、數值）

**重要事件：**
- `page_view` - 頁面瀏覽
- `scroll` - 滾動深度（25%、50%、75%、100%）
- `click` - 按鈕/連結點擊
- `submit` - 表單提交（留言）
- `page_time` - 頁面停留時間
- `page_load` - 頁面載入時間
- `login` - 登入行為
- `carousel_switch` / `carousel_click` - 輪播互動

### 2. 查看特定事件詳情
1. 在事件列表中點擊任一事件
2. 可以看到：
   - 事件觸發的時間趨勢
   - 事件參數（如 `event_label`、`event_category`、`value`）
   - 觸發事件的頁面
   - 觸發事件的用戶

## 三、查看頁面瀏覽資料

### 1. 頁面和畫面報表
**路徑**：「報表」→ 「參與度」→ 「頁面和畫面」

**可以看到：**
- 每個頁面的瀏覽次數
- 平均停留時間
- 跳出率
- 使用者數

### 2. 查看特定頁面的詳細資料
1. 點擊任一頁面
2. 可以看到：
   - 該頁面的流量來源
   - 用戶在該頁面的行為
   - 從該頁面前往的其他頁面

## 四、查看使用者行為

### 1. 使用者探索
**路徑**：「報表」→ 「探索」→ 「使用者探索」

**可以看到：**
- 使用者 ID（如果已設定）
- 每個使用者的瀏覽路徑
- 使用者的行為序列

**設定步驟：**
1. 建立新的探索報表
2. 選擇「使用者探索」
3. 在「維度」中加入「使用者 ID」
4. 在「指標」中加入想要查看的事件

### 2. 路徑探索
**路徑**：「報表」→ 「探索」→ 「路徑探索」

**可以看到：**
- 用戶從進入網站到離開的完整路徑
- 最常見的瀏覽流程
- 用戶在哪些頁面停留最久

## 五、查看特定追蹤項目

### 1. 滾動深度
**查看方式：**
1. 「報表」→ 「參與度」→ 「事件」
2. 搜尋「scroll」
3. 點擊「scroll」事件
4. 在「事件參數」中查看 `event_label`（會顯示 25%、50%、75%、100%）

**分析重點：**
- 有多少用戶滾動到 25%、50%、75%、100%
- 哪些頁面的滾動深度最高

### 2. 按鈕點擊
**查看方式：**
1. 「報表」→ 「參與度」→ 「事件」
2. 搜尋「click」
3. 點擊「click」事件
4. 在「事件參數」中查看 `event_label`（會顯示按鈕名稱）
5. 在「事件參數」中查看 `event_category`（會顯示類別：navigation、external_link、carousel 等）

**分析重點：**
- 哪些按鈕最常被點擊
- 外部連結的點擊率
- 導覽連結的使用情況

### 3. 頁面停留時間
**查看方式：**
1. 「報表」→ 「參與度」→ 「事件」
2. 搜尋「page_time」
3. 點擊「page_time」事件
4. 在「事件參數」中查看 `value`（停留秒數）
5. 在「事件參數」中查看 `event_label`（頁面路徑）

**分析重點：**
- 每個頁面的平均停留時間
- 哪些頁面最吸引用戶

### 4. 頁面載入時間
**查看方式：**
1. 「報表」→ 「參與度」→ 「事件」
2. 搜尋「page_load」
3. 點擊「page_load」事件
4. 在「事件參數」中查看 `event_label`（total_load_time、dom_content_loaded、first_paint）
5. 在「事件參數」中查看 `value`（載入時間，單位：毫秒）

**分析重點：**
- 頁面載入速度
- 哪些頁面載入較慢，需要優化

### 5. 登入行為
**查看方式：**
1. 「報表」→ 「參與度」→ 「事件」
2. 搜尋「login」
3. 點擊「login」事件
4. 在「事件參數」中查看 `event_label`（success 或 failure）

**分析重點：**
- 登入成功/失敗次數
- 登入失敗的原因（如果有記錄）

### 6. 留言提交
**查看方式：**
1. 「報表」→ 「參與度」→ 「事件」
2. 搜尋「submit」
3. 點擊「submit」事件
4. 在「事件參數」中查看 `event_label`（公開留言/不公開留言）

**分析重點：**
- 留言提交的頻率
- 公開 vs 不公開留言的比例

### 7. 輪播互動
**查看方式：**
1. 「報表」→ 「參與度」→ 「事件」
2. 搜尋「carousel」
3. 可以看到兩個事件：
   - `carousel_switch` - 自動切換
   - `carousel_click` - 手動點擊指示器
4. 在「事件參數」中查看 `value`（圖片編號）

**分析重點：**
- 用戶是否主動與輪播互動
- 哪些圖片最受關注

## 六、建立自訂報表

### 1. 建立探索報表
**路徑**：「報表」→ 「探索」→ 「空白」

**可以建立：**
- 自訂的資料視覺化
- 特定事件的分析
- 使用者行為分析

### 2. 建立轉換事件
**路徑**：「設定」→ 「轉換事件」

**可以將重要事件標記為轉換：**
- 報名網站點擊
- 留言提交
- 登入成功

**好處：**
- 可以追蹤轉換率
- 可以建立轉換漏斗

### 3. 建立受眾
**路徑**：「設定」→ 「受眾」

**可以建立：**
- 深度參與用戶（滾動到 100%）
- 高價值用戶（多次訪問）
- 管理員用戶（使用 User ID）

## 七、匯出資料

### 1. 匯出報表
1. 在任何報表中，點擊右上角的「匯出」按鈕
2. 可以匯出為：
   - PDF
   - Google Sheets
   - CSV

### 2. 使用 BigQuery（進階）
如果需要更深入的資料分析，可以：
1. 在 GA4 設定中啟用 BigQuery 匯出
2. 使用 SQL 查詢原始資料
3. 建立更複雜的分析

## 八、常用查詢範例

### 1. 查看今天的所有事件
1. 「報表」→ 「參與度」→ 「事件」
2. 在日期範圍中選擇「今天」
3. 查看所有事件的觸發次數

### 2. 查看特定頁面的事件
1. 「報表」→ 「參與度」→ 「事件」
2. 點擊任一事件
3. 在「次要維度」中選擇「頁面標題」或「頁面路徑」
4. 可以看到該事件在哪些頁面觸發

### 3. 查看使用者流程
1. 「報表」→ 「探索」→ 「路徑探索」
2. 選擇起始頁面（如首頁）
3. 可以看到用戶從首頁開始的瀏覽路徑

### 4. 比較不同時期的資料
1. 在任何報表中，點擊日期範圍
2. 選擇「比較」
3. 選擇要比較的時期
4. 可以看到資料的變化趨勢

## 九、注意事項

1. **資料延遲**：
   - 即時報表：幾秒鐘內可見
   - 標準報表：可能需要 24-48 小時才會完整顯示

2. **資料保留**：
   - 免費版 GA4：資料保留 14 個月
   - 付費版 GA4：可以設定更長的保留期

3. **隱私權**：
   - 某些瀏覽器擴充功能（如 AdBlock）可能會阻擋追蹤
   - 用戶可以選擇退出追蹤

4. **測試環境**：
   - 建議在生產環境測試
   - 本地開發環境可能無法正確追蹤

## 十、快速檢查清單

- [ ] 即時報表是否顯示訪客？
- [ ] 事件報表中是否有 `page_view` 事件？
- [ ] 事件報表中是否有自訂事件（如 `click`、`scroll`）？
- [ ] 頁面和畫面報表中是否有頁面瀏覽資料？
- [ ] 使用者探索中是否可以看到使用者 ID（如果已登入）？
